VPN Metadata Leakage Analysis

COMPLETE

Discovered VPN doesn't actually encrypt everything. Local network protocols (mDNS, Bonjour) bypass tunnel via split tunneling. Ran bettercap, saw Spotify/AirPlay traffic from phone despite "secure" WireGuard connection. Oops.

► Internet traffic encrypted. Local discovery protocols? Not so much.
► Device fingerprinting via local metadata. Privacy theater at its finest.

Social Engineering Toolkit Proxy Behavior

COMPLETE

SET doesn't clone websites. It proxies them. "Forgot password" links work, cookies captured, full session maintained. Way more sophisticated than I expected. Also broke on Python 3.13 because cgi.escape was removed. Classic.

► Real-time session hijacking, not static phishing
► Python 3.13 broke it immediately (deprecated imports)

Password Generation Pattern Analysis

COMPLETE

Accidentally created password (m4cus3r) that broke CUPP. Generated 1451 variations, missed the obvious one. Got M4cUser and M4cU53r (capitalized) but not lowercase + selective l33tspeak. Template-based generators have gaps.

► CUPP doesn't apply l33tspeak consistently across case variations
► Lesson: random inconsistency defeats pattern-based cracking

LESSONS LEARNED

► VPNs don't encrypt local network protocols (mDNS, Bonjour leak everything)
► Split tunneling is default on most VPNs (internet encrypted, LAN exposed)
► Device fingerprinting works even through VPN (MAC addresses, local metadata)
► Perfect privacy doesn't exist, only layers of inconvenience for adversaries
