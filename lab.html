LESSONS LEARNED

► Storage policy matters (default provisioner killed flash drives before dedicated StorageClass)
► Node naming convention prevents catastrophic mistakes (KittyKernel != PandaPayload)
► Frankencluster teaches resource heterogeneity better than cloud abstractions

RESEARCH PHILOSOPHY

This lab exists to understand how distributed systems, security mechanisms, and privacy tools work in practice. All experiments run on isolated infrastructure I own. Breaking your own things to learn how they work is education. Breaking other people's things is crime.

Focus: adversarial systems research, distributed computing patterns, privacy-preserving technology. Documenting failures because that's where the learning happens. Building proof-of-concepts for novel applications nobody asked for but might be useful.

► All targets are owned hardware within isolated network
► Open-source tools preferred for reproducibility
► Documentation published for educational benefit
► Research conducted for defense improvement, not attack development
