LLM OPTIMIZATION

ACTIVE PERSONAL

Local LLM optimization. Went from 3 tok/s to 25 tok/s (8.3x speedup) via Vulkan backend + speculative decoding. Running qwen2.5-coder-32b abliterated (safety removed). Fast enough to be useful, slow enough to be annoying.

PERFORMANCE GAINS

8.3x Total Speedup 3 → 25 tokens/second
Vulkan Backend AMD GPU optimization
14B + 0.5B Speculative Decoding Draft model acceleration

OPTIMIZATION PATH (chronological)

Step 1: Backend Testing Vulkan > ROCm for AMD consumer GPUs (better stability, equivalent speed)
Step 2: Speculative Decoding Qwen 2.5 14B + 0.5B draft model (major speedup with minimal quality loss)
Step 3: Configuration Tuning Context size optimization, cache settings, threading parameters
Step 4: Abliterated Models Safety-removed variants for uncensored security research (no refusals)
Result 25+ tokens/sec on consumer hardware (AMD GPU, no datacenter equipment)

► Hardware Optimization

AMD GPU tuning, VRAM allocation, thermal management. Tried not to melt the card. Mostly succeeded.

► Model Selection

Abliterated variants (uncensored for research), quantization testing (q4 vs q6). Quality vs speed trade-offs documented.

► Deployment Strategy

LM Studio with OpenAI-compatible API. Multi-model management, context optimization. Works well enough.

► Future Upgrades

Planned: 9950X3D, dual GPU (7800 XT + 7900 XTX), 96GB DDR5-6000, 2TB NVMe Gen5. £2,500 budget. Financially irresponsible but technically necessary.
